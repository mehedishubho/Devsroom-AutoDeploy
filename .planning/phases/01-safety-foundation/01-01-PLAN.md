**Schema changes (database/class-schema.php):**

1. In the `create_tables()` method, add two columns to the `devsroom_repositories` table SQL:
   - `locked_at` TIMESTAMP NULL DEFAULT NULL — when the lock was acquired
   - `locked_by` BIGINT(20) UNSIGNED NULL DEFAULT NULL — deployment ID that holds the lock
   Add these after the existing `updated_at` column in the CREATE TABLE statement.

2. Add a schema upgrade method `upgrade_schema()` that adds the columns if they don't exist:
   ```php
   public static function upgrade_schema(): void
   {
       global $wpdb;
       $table = $wpdb->prefix . 'devsroom_repositories';
       $columns = $wpdb->get_col("DESCRIBE {$table}");
       if (! in_array('locked_at', $columns, true)) {
           $wpdb->query("ALTER TABLE {$table} ADD COLUMN locked_at TIMESTAMP NULL DEFAULT NULL AFTER updated_at");
       }
       if (! in_array('locked_by', $columns, true)) {
           $wpdb->query("ALTER TABLE {$table} ADD COLUMN locked_by BIGINT(20) UNSIGNED NULL DEFAULT NULL AFTER locked_at");
       }
   }
   ```

**Lock methods (core/class-deployment-manager.php):**

3. Add method `acquire_lock(int $repository_id, int $deployment_id): array|false`:
   - Execute: `$wpdb->query($wpdb->prepare("UPDATE {$table} SET locked_at = NOW(), locked_by = %d WHERE id = %d AND locked_at IS NULL", $deployment_id, $repository_id))`
   - Check `$wpdb->rows_affected === 1` — if true, return `array('success' => true, 'message' => 'Lock acquired')`
   - If rows_affected === 0, check for stale lock: `$wpdb->get_row($wpdb->prepare("SELECT locked_at, locked_by FROM {$table} WHERE id = %d AND locked_at < DATE_SUB(NOW(), INTERVAL 10 MINUTE)", $repository_id))`
   - If stale lock found, force-acquire: `$wpdb->query($wpdb->prepare("UPDATE {$table} SET locked_at = NOW(), locked_by = %d WHERE id = %d", $deployment_id, $repository_id))` and return success with `'stale_lock_cleared' => true` in the array
   - If lock exists and NOT stale, return `array('success' => false, 'message' => 'Deployment already in progress for this plugin')` (per D-01)
   - Log all lock events via `$this->logger->info()` or `$this->logger->warning()`

4. Add method `release_lock(int $repository_id): void`:
   - Execute: `$wpdb->update($table_name, array('locked_at' => null, 'locked_by' => null), array('id' => $repository_id))`
   - No return value needed — called in finally block later

5. Add method `is_locked(int $repository_id): array|false`:
   - Query: `SELECT locked_at, locked_by FROM {$table} WHERE id = %d AND locked_at IS NOT NULL`
   - Return the row if locked, `false` if not locked

6. In the `deploy()` method, add lock acquisition as the FIRST operation after getting the repository:
   - After the existing repository lookup (around line 89), call `$this->acquire_lock($repository_id, $deployment_id)`
   - If lock acquisition fails (`$result['success'] === false`), update deployment status to `'failed'`, log the failure, and return `array('success' => false, 'message' => $result['message'])`
   - Store the lock status for later release
